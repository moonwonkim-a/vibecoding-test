# 도서 대여 관리 시스템 요구사항

## 1. 프로젝트 목적

이 프로젝트는 도서 대여 관리 시스템을 요구사항 문서 기반으로 구현하는 미니 프로젝트이다.

Cursor는 본 문서와 다음 문서를 기준으로만 구현한다.

- `architecture.md`
- `database-schema.md`
- `api-spec.md`
- `test-plan.md`
- `.cursor/rules/project-rules.mdc`

## 2. 화면 구성

웹은 총 2페이지로 구성한다.

### 2.1 이용자 페이지

일반 이용자가 도서 조회, 검색, 상세 확인, 대여, 반납, 대여 순위 조회를 수행하는 페이지이다.

주요 기능:

- 도서 목록 조회
- 도서 검색, 필터, 정렬
- 도서 상세 모달
- 도서 대여
- 도서 반납
- 대여 순위 TOP 10 조회

### 2.2 관리자 대시보드 페이지

관리자가 로그인 후 도서 관리, 대여 현황 조회, 반납 이력 조회, 불량 이용자 관리를 수행하는 페이지이다.

주요 기능:

- 관리자 로그인
- 전체 도서 및 재고 현황 조회
- 도서 추가
- 도서 삭제
- 대여 중 도서 목록 조회
- 이용자 반납 이력 조회
- 반납 이력 기반 불량 이용자 지정
- 불량 이용자 목록 조회 및 해제

## 3. 용어 정의

| 용어 | 정의 |
|---|---|
| 일반 이용자 | 도서를 검색, 열람, 대여, 반납하는 사용자 |
| 관리자 | 시스템 관리 권한을 가진 사용자 |
| 도서 마스터 | ISBN 기준 도서 메타데이터 |
| 실물 재고 | 도서관이 보유한 실제 책 1권 단위의 재고 |
| 대여 가능 도서 | 대여 가능한 실물 재고가 1권 이상 있는 도서 |
| 불량 이용자 | 연체, 분실, 훼손 등 사유로 대여가 제한된 이용자 |
| user_code7 | 테스트용 7자리 이용자 식별 코드 |
| 이용자 상태 정보 | 첫 대여 시 자동 생성되는 이용자 식별/불량 상태 정보 |

## 4. 개인정보 처리 원칙

요구사항 원문에는 주민번호 앞 7자리 표현이 있으나, 구현에서는 실제 주민번호를 수집하거나 저장하지 않는다.

구현에서는 `user_code7`이라는 테스트용 7자리 이용자 식별 코드를 사용한다.

규칙:

- `user_code7`은 숫자 7자리 문자열이다.
- 실제 개인정보를 의미하지 않는다.
- 화면 안내 문구도 테스트용 식별 코드로 표현한다.
- 관리자 화면에서는 `user_code7`을 앞 4자리 + `***` 형태로 마스킹 표시한다.
- 일반 이용자는 사전 회원가입을 하지 않는다.
- 이용자가 처음 대여할 때 입력한 이름과 `user_code7`을 기준으로 이용자 상태 정보가 자동 생성된다.

## 5. 이용자 기능 요구사항

### FR-LIST: 도서 목록

| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-LIST-001 | 도서 목록은 ISBN 기준 1행으로 표시하고 30건 단위 페이징을 제공한다. | 필수 |
| FR-LIST-002 | 각 행에 카테고리, 저자, 출판사, 제목, 가격, 대여 가능 현황, 상세 버튼을 표시한다. | 필수 |
| FR-LIST-003 | 제목, 저자, 카테고리, 가격, 대여 가능 여부 기준 정렬을 제공한다. | 필수 |
| FR-LIST-004 | 검색어와 검색 필터를 제공한다. 필터는 제목, 저자, 카테고리, 출판사이다. | 필수 |
| FR-LIST-005 | 대여 가능 재고가 0권인 도서는 대여 불가 상태로 시각 구분한다. | 필수 |

### FR-DETAIL: 도서 상세

| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-DETAIL-001 | 상세 버튼 클릭 시 중앙 모달을 표시한다. | 필수 |
| FR-DETAIL-002 | 모달에 카테고리, 저자, 출판사, 제목, 가격, 총 보유 수량, 대여 가능 수량을 표시한다. | 필수 |
| FR-DETAIL-003 | 대여 가능 재고가 1권 이상이면 대여 버튼을 활성화한다. | 필수 |
| FR-DETAIL-004 | 대여 가능 재고가 0권이면 대여 버튼을 비활성화하고 안내 문구를 표시한다. | 필수 |
| FR-DETAIL-005 | 모달 외부 클릭 또는 닫기 버튼으로 모달을 닫을 수 있다. | 필수 |

### FR-RENT: 도서 대여

| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-RENT-001 | 대여 창에서 이름과 `user_code7`을 입력받는다. | 필수 |
| FR-RENT-002 | 입력 후 해당 이용자의 현재 대여 중 도서 목록을 조회하여 표시한다. 신규 이용자는 현재 대여 0권으로 처리한다. | 필수 |
| FR-RENT-003 | 현재 2권 대여 중인 이용자는 추가 대여할 수 없다. | 필수 |
| FR-RENT-004 | 불량 이용자는 대여할 수 없다. | 필수 |
| FR-RENT-005 | 대여 가능 조건을 만족하면 대여 버튼을 활성화한다. | 필수 |
| FR-RENT-006 | 대여 시 서버가 대여 가능한 실물 재고 1건을 자동 할당한다. 이용자 상태 정보가 없으면 자동 생성한다. | 필수 |
| FR-RENT-007 | 할당된 재고는 `available=false`, `rent_date=오늘`로 변경한다. | 필수 |
| FR-RENT-008 | 대여 기록을 생성하고 반납 기한은 대여일 + 21일로 저장한다. | 필수 |
| FR-RENT-009 | 대여 완료 후 반납 기한을 안내한다. | 필수 |

| EX-008 | 관리자 로그인 실패 | 접근 거부 | 아이디 또는 비밀번호가 올바르지 않습니다. |

| EX-014 | `user_code7` 형식 오류 | 입력 차단 | 이용자 식별 코드는 숫자 7자리여야 합니다. |
